# Research Questions: Rework README to professional standard with clean setup section

## Discovery Summary

Three parallel discovery agents mapped the setup surface area and the family-wide README conventions. The setup flow is implemented entirely in `extensions/rpiv-core/index.ts` (243 lines) with three supporting pure-Node modules: `package-checks.ts` (regex detection against `~/.pi/agent/settings.json`), `agents.ts` (bundled-agent auto-copy), and `guidance.ts` (guidance injection). Five sibling plugins are hard-required and detected at session start: `@tintinweb/pi-subagents`, `@juicesharp/rpiv-ask-user-question`, `@juicesharp/rpiv-todo`, `@juicesharp/rpiv-advisor`, `@juicesharp/rpiv-web-tools` — each listed in `peerDependencies` at `package.json:11-21`.

Sibling READMEs (advisor, ask-user-question, todo, web-tools) share a tight convention: one-paragraph lede under title, `docs/*.jpg` screenshot, 4-space-indented `pi install npm:@juicesharp/…` block, "Then restart your Pi session." trailer, and `## License\n\nMIT`. The parent `rpiv-pi` README breaks most of these: no lede, no screenshot, no License; uses fenced `bash` blocks and a `Version: 0.4.0` line. `rpiv-skillbased/README.md` (same family, titled `rpiv-next`) is the richest long-form reference and uses a `/rpiv-next:<name>` skill prefix — different from the parent's `/skill:<name>` — an inconsistency the rework must resolve.

Eight prior thoughts docs cover the 0.3.x → 0.4.0 plugin extraction (just shipped in commit `c388ea9`) and the original Pi migration. Authoritative decisions to respect: `peerDependencies` + runtime `session_start` check + `/rpiv-setup` hard-fail pattern; no mention of `pi-permission-system` in user-facing READMEs; "Migration from 0.3.x" section must reference all four new `@juicesharp/rpiv-*` plugin names; Brave key resolves env-var-first then `~/.config/rpiv-web-tools/config.json` (0600); silent config-path cutover for advisor/web-tools.

Key files read for depth: `README.md`, `package.json`, `extensions/rpiv-core/index.ts`, `extensions/rpiv-core/package-checks.ts`, and the four sibling READMEs (`rpiv-web-tools`, `rpiv-advisor`, `rpiv-todo`, `rpiv-skillbased`).
